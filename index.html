This template is used to demonstrate a hand-built, responsive webpage that dynamically alters menus, layout, and content depending on the dimensions of the browser window.

Content 1

This box is used for the primary content which will be displayed in all versions of the website. On a browser window that is wide enough, this box will be on the left, with the less important box content2 placed on the right side of the browser window.

  1. If you want to adjust the relative width of the boxes, that can be done via CSS.
  2. If you want to change the colors, etc., that can all (obviously) be accomplished via the CSS.
  3. Right now there's a "container" div that is used to leave whitespace margins on either side of the page. That can be removed as well, allowing for more content on a single page.

To view how CSS alters the display of these files based on browser window width, load them into a web browser and then resize the window from wide to narrow and back.

This is Content 2

In this template, the content2 div is placed on the right in the desktop version. When the window shrinks to a size that doesn't easily allow for the side-by-side placement of these two divs, content2 is dropped down below content1 in the viewport.

There is also the option for removing content entirely from the mobile view by tagging a block, img, paragraph, etc, with the 

class="mobile_hide"
class. In this example, there is an image that is displayed in the desktop view, but that image is not displayed in the mobile version.
